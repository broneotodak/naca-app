About the numbering

Numbering matches the spec exactly: docs/spec/platform-refactor-v1.md. Sub-labels like 2b and 4a / 4b exist because the plan evolved mid-refactor — Phase 2 swapped with Phase 3 (tools depend on core, so core had to go first); Phase 4 split into 4a (the abstract engine, done) and 4b (de-Siti of rules + permissions + 8 specialists, pending) once we saw how WhatsApp-coupled the specialists are. Both spec doc and this page now agree.

Why this refactor

The one-paragraph version

Every tool, router rule, and specialist used to live inside siti-v2. That meant any new NACA interface — the mobile app, an MCP bridge for Claude Code sessions, future voice surfaces — would need to duplicate the same code or import from a WhatsApp repo. The refactor lifts the platform code (kernel, capabilities, router engine) into standalone @naca/* packages that any interface can consume. Siti stays as the WhatsApp interface; the platform stays platform.

4b

Router rules + permissions + specialists — closed as no-op

Original plan was to lift rules.js, permissions.js, and 8 specialists out of siti-v2 into @naca/router. Same outcome as Phase 2b: the premise didn't hold on inspection. Rule definitions are WhatsApp/BM-English text patterns (other interfaces don't have natural-language input). Permissions queries the WA-operational contacts table with JID normalization (naca-app uses mobile user IDs; MCP uses bearer tokens). Specialist business logic differs per interface (CHAT for WA ≠ CHAT for mobile ≠ CHAT for MCP). siti-v2's rules/permissions/specialists are correctly the WhatsApp adapter consuming the engine that Phase 4a already lifted. Future interfaces build their own adapters on the same engine. Closed as no-op.

Glossary

Names that matter
NACA
Neo's Agentic Centre Applications. The whole platform.
@naca/core
Shared kernel package. Postgrest, vault, heartbeat, Gemini clients, abstract send.
@naca/tools
Capability library — the 25 tools any NACA interface can dispatch.
@naca/router
Intent classifier engine + LLM tiebreak. Knows nothing about WhatsApp.
siti-v2
The WhatsApp interface. Now imports @naca/core + @naca/router + @naca/tools instead of carrying its own copies.
Interface
Any app that uses NACA. Currently siti-v2 (WhatsApp). Soon naca-app (mobile), naca-mcp-bridge (Claude Code).
neo-brain
The Supabase project (xsunmervpyrplzarebva) that holds memory, contacts, agent state. Accessed via @todak/memory SDK (lives in CTK, not NACA).
