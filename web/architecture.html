02The seven layers

Bottom to top. Each layer has one clear job.

1

🧠 neo-brain

One Postgres + pgvector database. Memory (hybrid RAG retrieval) and the fleet's operational data.

2

📚 SDKs — @todak/memory, @naca/core

The code everything uses to talk to neo-brain. Never raw SQL.

3

🧰 @naca/tools

~25 capabilities — the verbs: search memory, send email, read a URL, generate an image.

4

🔌 MCP

Exposes the tools so any AI client can call them — without custom wiring per client.

5

🧭 Router

Reads an incoming message, classifies the intent, routes it to the right specialist.

6

🚪 Interfaces

The front doors people talk to — Siti on WhatsApp, the NACA app. (Some setups call these "channels.")

7

🤖 Agents

The autonomous fleet — planner, reviewer, dev, verifier, timekeeper, supervisor. They do work unattended. This layer is the autonomy step — the difference between "an AI you ask" and "a crew that works while you sleep."

03Jargon, decoded

The words that make this sound harder than it is.

API — two programs talking over the network.
SDK — pre-built plumbing code so you don't rewrite it everywhere.
MCP — a universal plug; lets any AI client use our tools.
Router — the receptionist; routes a message to the right specialist.
Agent — a worker that runs 24/7 on its own; nobody pokes it.
Interface / Channel — a front door (WhatsApp, the app).
CTK — the workshop: internal dev toolkit. It builds the product; it isn't the product.
neo-brain — the one memory + operational database.

04How a request actually flows

Nothing abstract — one real example, start to finish.

Neo WhatsApps Siti: "remind me to call Lan at 5pm."

Siti (interface) receives the message.

The Router classifies it — intent: REMINDER.

The timekeeper agent schedules it — a row in neo-brain, written via the SDK.

At 5pm, timekeeper fires, calls the send-message tool, and Siti delivers the reminder.

No human touched steps 2–4. That is the fleet.
